# 首次启动说明

这一步只做一件事，先把服务跑起来。

别一上来就处理 license。先把容器启动、健康检查和控制台访问跑通，再决定要不要启用增强版。

## 1. 准备目录

仓库根目录里已经预留了两个目录：
- `./data`

首次启动时，核心会用到的是：
- `./data`

## 2. 检查 [compose.yaml](../compose.yaml)

默认情况下你不需要额外创建 `.env` 文件。

如果你只想先跑起来，通常只要确认两件事：
1. `image` 是否保持默认值
2. `ports` 是否和你机器上的端口冲突

如果你想把端口改成 `8040`，直接把：

```yaml
- "8000:8000"
```

改成：

```yaml
- "8040:8000"
```

## 3. 启动

```bash
docker compose up -d
```

## 4. 验证

先看容器状态：

```bash
docker compose ps
```

再看健康检查：

```bash
curl http://127.0.0.1:8000/api/healthz
```

如果你改过端口，把命令里的 `8000` 一起改掉。

成功返回：

```json
{"status":"ok"}
```

## 5. 打开控制台

默认地址：
- <http://127.0.0.1:8000/app>

如果你改过端口，就访问对应的新端口。

## 6. 这一阶段你应该确认什么

先确认这四件事：
1. 容器能正常启动。
2. `/api/healthz` 能返回正常状态。
3. 控制台能打开。
4. 你已经确认这套部署形态符合自己的使用场景。

这四件事跑通后，再去看增强版。
